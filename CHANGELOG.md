### v0.5.5 (2026-04-22)

- **新增顶层硬约束：事实发明明令禁止**。在「⚠️ 信息完整性」章节补「🚫 硬约束：不得在终稿中发明原文没有的具体事实」子节：禁止发明具体数字、事件、不存在的主体、金额时间地点、引用原话。"注入人味"的合法路径收窄为一条：只能从原文已有信息里挑、提炼、具体化,或以明显的作者视角补注,不允许虚构。
- **新增顶层硬约束：门检强制输出格式**。在「🛑 第负一步」节补「✅ 门检必须显式输出判断声明」：任何改写工作之前都要输出一行 `【门检】判断：AI 生成文本 / 真人文本（停手） / 不确定 | 证据：[≤2 条具体信号]`,证据要写原文里的具体词 / 短语,不写抽象判断。
- **新增顶层硬约束：语体降级保护**。在「⚠️ 避免过度修正」节补「语体降级保护」子节,明确五档语体档位（公文 / 法律 > 学术 / 科技 > 商务对外正式 > 书面 / 一般 · 自媒体 > 商务内部 · 朋友圈），改写只允许在相邻档之间切换,不得跨档。
- **AI 不敢写测试补「原文核查」条款**。新增「⚠️ 额外核查：发明的毛边不算通过」子节,与「信息完整性」硬约束互锁:正向检测（终稿里有真人迹象）+ 反向禁令（终稿里没有原文没有的事实）,两者都过才合格。
- **技术博客语体归档**。在「九种基础语体」节补技术博客 / 工程博客 / AI 科普文的归档判据:短段 < 300 字出现 ≥ 3 个英文技术全拼词 或 出现"中文（English full name）"首次引入格式,按「学术 / 科技」处理。英文白名单扩展:`latency` / `context` / `throughput` / `quantization` / `pruning` / `embedding` / `inference` / `attention` / `token` / `kernel` 等工程全拼术语与 `API / SDK / LLM / RAG` 同级保留英文,#42 不触发。
- **新增 WARP.md**,扩展 Warp 平台支持（跟 .cursorrules 同为 SKILL.md 的镜像版本,仅去掉 YAML frontmatter）。
- **新增 .gitignore**,公开仓库只留 README / LICENSE / SKILL.md / .cursorrules / WARP.md / .github/。
- 总规则数仍为 51。顶层硬约束由 2 条（过度消毒反制 + AI 不敢写测试）扩展为 **5 条**（+ 事实发明禁令 + 门检强制输出 + 语体降级保护）。

### v0.5.4 (2026-04-22)

- **重写规则 #16**：从"单一密度阈值 `≥3 in <200 字 且 Y 空转`"改为**语体分层触发**:
  - 日常 / 自媒体 / 社交 / 商务 / 邮件 / PRD: `<300 字 ≥2 次同位补注`即触发（不再要求空转）
  - 书面 / 一般: `<300 字 ≥3 次`触发
  - 杂文 / 特稿 / 叙事非虚构 / 品牌广告: 维持旧阈值
  - 学术 / 公文 / 法律 / 高考应试作文: 不触发
- **重写规则 #25**：同样语体分层,日常商务档`500 字 ≥2 处凑数破折号`即触发。**#16 只管同位补注形状、#25 只管纯频率凑数**,消除原有概念重叠。
- **两个规则共用逐处自查问**:`这个 —— 能否无损替换成 , ( : 。？` 能替换 → 凑数,改;不能 → 承担真实修辞功能,保留。
- **新增「破折号的密度基线」子节**（在「标点符号」章节内）。
- **保留示范豁免**：鲁迅 / 钱钟书 / 董桥 / 王小波的杂文破折号 + Apple / Nike 品牌广告节奏破折号（`这不是升级——是重来`）**禁改**。
- 总规则数仍为 51, 顶层硬约束仍为 2 条。

### v0.5.0 (2026-04-21)

- **新增顶层硬约束：终稿自检 · AI 不敢写测试**。不依赖原文,直接问终稿：有没有**任何一句**是 AI 不敢写的 —— 自嘲 / 承认不确定 / 冒犯性观点 / 身份降格 / 只有当事人才知道的具体细节？一句都没有 → 太干净,回去加一处。跟 `硬约束`（保留原有毛边）互补,两者都过才算改写合格。
