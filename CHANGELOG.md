### v0.6.3（2026-04-23）

**一致性与可执行性校准。** 本版不新增规则，不改变 51 条模式与 7 条硬约束的语义边界，重点是把容易造成歧义的表述收紧，降低误用成本。

- **规则计数统一。** 将文中残留的 `50` / `33` 统一为 `51`，包括语体激活说明、流程扫描说明、检测启发式说明、规则计数总览与相关提示。
- **门检分支显式化。** 新增「执行分支（先判停手再决定输出）」：先做第负一步门检，再分流到 A（真人停手）、B（AI 继续）、C（不确定先反问），补齐与「真人文本停手」原则的一致性。
- **输出格式改为门检先行。** 将输出规范改成「先门检声明，再按分支输出」，明确真人文本场景只给停手说明与可选格式清理，避免误触发完整改写流程。
- **标点对照表纠偏。** 修正半角/全角示例，并补 Unicode 码位（如 `,`/`，`、`:`/`：`、`?`/`？` 等），减少视觉近似导致的误判。
- **默认语体表述收紧。** 无样本时默认改为「书面 / 一般（中性书面）」，仅在用户明确要求时下调到日常中性白话，避免默认降格。
- **版本与镜像同步。** `SKILL.md` frontmatter 升至 `0.6.3`，并通过 `scripts/build-flat.sh` 同步 `.cursorrules` 与 `WARP.md`。

### v0.6.2（2026-04-22）

**结构重构。SKILL.md 从 2533 行拆成 1207 行核心 + 5 个 references/ 按需加载文件,行为不变,但减少"lost-in-the-middle"风险。**

v0.6.1 及之前,SKILL.md 已经涨到 2533 行(约 60-90K tokens)。skill 一被触发,整份文件就进入上下文,带来两个副作用:

- **lost-in-the-middle**:深埋在 2000 行以后的规则(#45-#48,#49-#50)在长改写里被模型跳过的概率上升;
- **上下文挤占**:改写长文章时,原文 + 对话本身要跟 90K tokens 的 SKILL.md 抢地方。

### v0.6.0（2026-04-22）

**合格线从「不像 AI 了」抬到「干净且精准」。** v0.5.x 只清显眼 AI 腔，常留下"干净的中性散文"：动词弱、节奏平、filler 扎堆，读着仍然平庸。v0.6.0 起，这个 skill 同时扮演**扫地机 + 打磨工**两角色 —— 不只删，也主动把动词、节奏、filler、语序、语体打磨到位，前提是**不虚构**。

**用户看得到的变化：**

- **skill 现在会主动打磨，不只是删 AI 腔**。六类 craft 工作：弱动词替换（`进行讨论` → `讨论`）、节奏重塑（打断机关枪式句长）、filler 切除（`一些 / 实际上 / 在一定程度上` 一类）、抽象换具体（**只从原文已有细节里挑**，不凭空发明）、语序归位、语体匹配。
- **每份终稿都附「打磨报告」**。列清楚动了哪里（动词 / 节奏 / filler / 抽象 / 语序 / 语体档位）+ 可观察指标（具体名词 / 第一人称 / 引用原话 / 段长变异）。报告里的**每处例子都是原文 / 终稿的字面引用**，不能是撰写范例。数量说不准就用"若干处 / 多处 / 通篇"，**不伪造整数**。报告只描述，不开药方 —— 不会跟你说"建议补充 XX"。
- **完成阈值抬高：从"一句有声口"到"全篇都过"**。v0.5.0 的「AI 不敢写测试」只要求终稿里一句是 AI 不敢写的。v0.6.0 加「整篇 craft 自检」五问（用词精准 / 节奏变化 / 动词驱动 / 具体替抽象 / 声音一致），两样都过才算合格。
- **诚实边界**：原文观点平庸的 → 出来是"干净的平庸"；观点清晰的 → 出来是"一份像样的稿子"；观点独到 + 细节具体的 → 接近可读的水位。把普通稿子改成锋利有声口的好文章 —— 不在能力范围。这条边界在 README 的「能做什么，不能做什么」章节明确讲清楚了。
- **事实发明禁令继续生效**。v0.5.5 立的"不得发明原文没有的事实"硬约束**优先级高于**本版 craft 抬升。craft 的合法路径是把原文已有的东西弄得更锋利，不是"发现原文隐藏的灵感"。

**实现层面：** 顶层硬约束从 5 条扩到 7 条（+ 主动打磨 + 整篇 craft 自检）。规则编号不变，总数仍为 51 —— 6 条 craft moves 是对现有规则 #12 / #30 / #31 / #33 / #34 / #35 的**主动姿态**，不是新规则。SKILL.md / .cursorrules / WARP.md / README 同步跟改，frontmatter 描述更新。

**已知限制：** v0.5.5 README 引用的 [`tests/integration-run/`](./tests/integration-run/) 目录未提交进 git，本版的 craft 抬升**没有**跑过该 regression 管道。v0.6.0 设计上**保留并强化**了 v0.5.5 的事实发明禁令（打磨报告的四条硬约束就是针对这类幻觉的二次封堵），但建议在下一版恢复 tests 目录之后，用已有样本 + 新增至少一段 craft 抬升场景样本，跑一轮完整 regression，验证 craft 不增加虚构。

### v0.5.6（2026-04-22）

**新增「标点符号的功能分工」第二层规范。** 此前 SKILL.md 的「标点符号」章节只覆盖"全角半角"和破折号密度基线（#16 / #25），对顿号 / 分号 / 书名号 / 省略号 / 冒号 / 引号 / 括号的功能分工未作系统说明。

- **新增 SKILL.md 章节「标点符号的功能分工（第二层规范）」**，在"全角半角"小节后、"语序"小节前。覆盖七个子小节：
  - 顿号 vs 逗号（并列项一律顿号）
  - **分号**（语体分层，社论 / 长议论里分号排比是合法节奏工具，非 AI 腔）
  - 书名号（覆盖刊物、政策文件、栏目、MV、年度产品等）
  - **省略号**（严肃正文基线 ≈ 0，比破折号还稀有）
  - 冒号 vs 破折号（引出信息是冒号的活，破折号承担的是补注 / 强调 / 转折）
  - 引号（直接引语 / 术语标引 / 反讽 三种合法用途）
  - 括号（次要补注，不引出主干）
- **修正一：分号。** 前版「真人用很少，高密度 = AI 腔」只适用于日常 / 商务 / 自媒体等非论述语体。在社论 / 时评 / 长议论 / 学术论证里，分号用于结构对称的排比句群分层，是合法修辞。判据：分号两侧是不是结构对称的句子？对称保留，不对称改。
- **修正二：省略号。** 严肃议论正文里省略号基线 ≈ 0，比破折号还稀有。省略号只在对话 / 口述实录 / 列举未完 / 营销文案 / 随意语体里出现；严肃议论正文里出现省略号反而是刺眼信号。
- **新增「严肃媒体标点频率基线」**：`。 , 、 : "" ? ! 《》 —— ; …… ( )` 从高频到稀有的排序，作为 AI 腔偏离度的校准证据。AI 输出里 `——` `;` `( )` 经常反常密集，`《》` 和 `……` 经常反常稀少。
- **.cursorrules / WARP.md 同步跟改**，保持三份文件一致（延续 v0.5.3 立下的 dogfooding 规矩）。
- 总规则数仍为 51，顶层硬约束仍为 5 条。本版是标点用法规范的扩展，不新增检测模式编号。

### v0.5.5（2026-04-22）

- **新增顶层硬约束：事实发明明令禁止**。在「⚠️ 信息完整性」章节补「🚫 硬约束：不得在终稿中发明原文没有的具体事实」子节：禁止发明具体数字、事件、不存在的主体、金额时间地点、引用原话。"注入人味"的合法路径收窄为一条：只能从原文已有信息里挑、提炼、具体化,或以明显的作者视角补注,不允许虚构。

- **新增顶层硬约束：语体降级保护**。在「⚠️ 避免过度修正」节补「语体降级保护」子节,明确五档语体档位（公文 / 法律 > 学术 / 科技 > 商务对外正式 > 书面 / 一般 · 自媒体 > 商务内部 · 朋友圈），改写只允许在相邻档之间切换,不得跨档。
